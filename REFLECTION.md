# REFLECTION

This project taught me that building software is not only about implementing features but also about making decisions when requirements are intentionally incomplete.

One of the biggest challenges was deciding what not to build. It was tempting to implement authentication, notifications and many additional features, but keeping the scope focused resulted in a more complete application.

Designing both the backend and the frontend also reinforced the importance of keeping APIs simple and consistent.

If I started again, I would define the API contracts before implementing the frontend. This would reduce the amount of integration work required later.

Throughout the project I also improved my understanding of Git workflows, incremental development and structuring a full-stack application into reusable components.

Overall, this project strengthened both my technical skills and my ability to make engineering decisions under uncertainty.
